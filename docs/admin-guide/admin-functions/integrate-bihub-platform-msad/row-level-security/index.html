Row Level Security for Power BI Reports

The Row Level Security feature has been implemented for the Power BI Reports. The steps followed for the configuration of the Row Level Security feature has been explained below for each mode:

  • Admin Mode
  • User Mode

Admin Mode#

  1. For Power BI Agent, once the Instance is being newly added, then the status will be in “Yellow” color in the Instance Manager screen (see Figure below). The Yellow color status indicates that the Authentication is pending for the Agent.

  2. Now select the Instance and click the "Login as Admin" button as shown in the below Figure.

  3. A pop up window appears where the Admin needs to provide the Power BI Administrator Login credentials (see Figure below).

  4. Now the user will be authenticated successfully as shown in the below Figure.

  5. After providing the login credentials, the Power BI Agent will be activated and now the status will be in “Green” color in the Instance Manager screen (see Figure below).

User Mode#

  1. After logging in as an User in BI Hub application, the user will be able to access the Power BI Reports (see Figure below).

    Power BI Report

  2. In our example, the VBI View Demo user has mapped the Power BI Agent "Power BI QE Test" with the user “powerbiagent” in the Agent Mapping screen in Admin Mode (see Figure below).

  3. Now when the VBI View Demo user selects the Power BI Report in the User screen (see Figure: Power BI Report) then he needs to provide the powerbiagent login credentials to access the Power BI Report (see Figure below).

  4. If the VBI View Demo user has provided any other user login credentials (other than powerbiagent), then an Error Message will be displayed as shown below.

  5. For the Row Level Security implementation in the Power BI Platform Reports, the Toggle button for the "Comply with Row Level Security" option in the Add Agent screen of the Admin Mode should be enabled (see Figure below).
